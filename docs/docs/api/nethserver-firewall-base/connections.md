# connections

Manage conntrack table.

## read

### Input

The read API requires an action field.
Valid actions:

- `conntrack`
- `protocols`
- `stats`

#### protocols

List the available protocols along with their states.

Example:
```json
{
  "action": "protocols"
}
```

#### conntrack

Return the list of connections filtered by `protocol` and `state`.

Example:
```json
{
  "action": "conntrack",
  "protocol": "tcp",
  "state" : "ESTABLISHED",
}
```

#### stats

Return the connection number in the last 30 seconds from netdata.

Example:
```json
{
  "action": "stats"
}
```

## delete

The read API requires an action field.
Valid actions:

- `flush`
- `connection`

### Input

#### flush

Delete all active connections.

Example:
```json
{
  "action": "flush"
}
```

#### connection

Delete selected connction.
Required fields are: `protocol`, `src`, `dst`.
If `protocol` is `udp` or `tcp`, also `sport` and `dport` are required.

Example:
```json
{
  "action": "connection",
  "protocol": "icmp",
  "dport": 389,
  "sport": 46286,
  "src": "10.0.0.212",
  "dst": "1.2.3.4"
}
```
